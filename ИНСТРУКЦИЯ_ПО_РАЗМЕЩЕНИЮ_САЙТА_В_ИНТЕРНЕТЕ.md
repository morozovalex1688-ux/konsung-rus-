## Что нужно сделать в итоге

К концу этой инструкции у вас будет:

- Рабочий сайт по адресу вида `https://konsung-rus.vercel.app` (бесплатно)
- Или по вашему домену `https://konsungrus.ru` (рекомендуется)
- Полностью рабочая форма обратной связи (заявки приходят на `info@konsungrus.ru`)
- Все PDF-инструкции скачиваются
- Яндекс.Карта работает
- Вы сможете самостоятельно менять цены, добавлять товары, править тексты и выкладывать изменения за 2–3 минуты

**Время на выполнение:** 40–90 минут (в зависимости от скорости интернета и того, есть ли у вас уже домен).

## Что понадобится (аккаунты и доступы)

| Что нужно | Где регистрироваться | Зачем | Обязательно? |
|-----------|----------------------|-------|--------------|
| **GitHub аккаунт** | github.com | Хранить код сайта | Да |

### 1.1. Откройте папку проекта в терминале

Должно вывести путь, заканчивающийся на `konsung-rus`.

## Часть 3. Создаём репозиторий на GitHub (самый важный шаг)

### 3.1. Создаём репозиторий

1. Зайдите на [https://github.com](https://github.com) (вы должны быть авторизованы)
2. В правом верхнем углу нажмите **+** → **New repository**
3. Заполните поля:

   - **Repository name**: `konsung-rus` (рекомендуется) или `konsungrus-website`
   - **Description** (опционально): `Официальный сайт ООО «Консунг Рус» — эксклюзивный дистрибьютор Sejoy в России`
   - **Public** — оставьте (бесплатно)
   - **НЕ ставьте галочку** "Add a README file"
   - **НЕ ставьте галочку** "Add .gitignore"
   - **НЕ ставьте галочку** "Choose a license"

4. Нажмите большую зелёную кнопку **"Create repository"**

5. На следующей странице вы увидите инструкцию "Quick setup".  
   **Скопируйте HTTPS-ссылку** — она выглядит так:
   ```
   https://github.com/ВАШ_ЛОГИН/konsung-rus.git
   ```

### 3.2. Подключаем репозиторий к вашему компьютеру

Вернитесь в терминал (в папке проекта) и выполните команды **по очереди** (замените ссылку на свою!):

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/konsung-rus.git
git branch -M main
git push -u origin main
```

При первом пуше GitHub попросит логин и пароль (или Personal Access Token).

**Как ввести пароль:**
- Когда терминал попросит `Password for 'https://github.com'`, вставьте **Personal Access Token**, а не пароль от GitHub.
- Как создать токен: GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token → поставьте галочку `repo` → Generate → скопируйте.

После успешного пуша вы увидите примерно:

```
Enumerating objects: 85, done.
...
To https://github.com/ВАШ_ЛОГИН/konsung-rus.git
 * [new branch]      main -> main
Branch 'main' set up to track remote branch 'main' from 'origin'.
```

**Поздравляем!** Код сайта теперь хранится на GitHub.

## Часть 4. Первый деплой на Vercel

### 4.1. Импортируем проект

1. Откройте [https://vercel.com](https://vercel.com) и войдите
2. На главной странице нажмите большую синюю кнопку **"Add New Project"**
3. Вы увидите список ваших GitHub-репозиториев
4. Найдите `konsung-rus` и нажмите кнопку **"Import"** справа от него

### 4.2. Настройки проекта (почти всё можно оставить по умолчанию)

На экране "Configure Project" проверьте:

- **Framework Preset** — должен автоматически определиться как **Next.js** (если нет — выберите вручную)
- **Root Directory** — оставьте пустым (или `.`)
- **Build Command** — оставьте пустым (Vercel сам подставит `next build`)
- **Output Directory** — оставьте пустым (`.next`)

Нажмите **"Deploy"** внизу.

### 4.3. Ждём первый деплой (1–3 минуты)

Вы увидите красивый лог сборки. Это нормально:

```
Cloning github.com/...
Installing dependencies...
Detected Next.js...
Building with Next.js 16...
...
Build Completed
```

В конце появится зелёная надпись:

**"Your site is live!"**  
и большая кнопка **"Visit"**

Нажмите **Visit** — откроется ваш сайт по адресу вида:
`https://konsung-rus-abc123.vercel.app`

**Сохраните этот адрес!** Это пока ваш основной рабочий сайт.

## Часть 5. Настройка отправки писем (Resend) — обязательно!

Без этого шага форма на сайте **не будет работать**.

### 5.1. Добавляем переменную окружения в Vercel

1. В дашборде Vercel откройте ваш проект `konsung-rus`
2. Перейдите во вкладку **Settings** (вверху)
3. В левом меню выберите **Environment Variables**
4. Нажмите **"Add New"**

Заполните:

- **Name**: `RESEND_API_KEY` (точно так, без пробелов)
- **Value**: вставьте ваш ключ от Resend (`re_xxxxxxxxxxxxxxxxxxxxxxxx`)
- **Environment**: поставьте галочки на все три варианта:
  - Production
  - Preview
  - Development

5. Нажмите **Save**

### 5.2. Передеплоиваем проект

После сохранения переменной:

1. Перейдите во вкладку **Deployments** (вверху)
2. Найдите самый верхний (последний) деплой
3. Нажмите на три точки `⋯` справа → **Redeploy**
4. Подтвердите

Или проще: на главной странице проекта нажмите **"Redeploy"** в правом верхнем углу.

Подождите 1–2 минуты, пока появится новая зелёная галочка.

### 5.3. Проверяем, что письма работают

1. Откройте ваш сайт (vercel.app адрес)
2. Пролистайте до раздела **Контакты**
3. Заполните форму тестовыми данными:
   - ФИО: `Тест Тестов`
   - Телефон: `+7 999 123-45-67`
   - Email: ваш личный email (чтобы проверить)
   - Компания: `Тестовая аптека`
   - Сообщение: `Это тестовая заявка. Можно удалить.`
4. Поставьте галочку согласия и нажмите **"Отправить заявку"**

Через 10–30 секунд проверьте почту `info@konsungrus.ru` (и папку Спам на всякий случай).

Если письмо пришло — **всё работает отлично**.

Если не пришло — смотрите раздел Troubleshooting.

## Часть 7. Подключаем свой домен (konsungrus.ru)

Это необязательно, но сильно повышает доверие.

### 7.1. Добавляем домен в Vercel

1. В проекте на Vercel перейдите **Settings → Domains**
2. В поле "Add Domain" введите `konsungrus.ru` (без www)
3. Нажмите **Add**
4. Vercel покажет вам, какие DNS-записи нужно добавить.

Обычно это:

**Вариант A (рекомендуется для apex-домена):**
- Тип: `A`
- Имя: `@` (или пусто)
- Значение: один из IP-адресов Vercel (Vercel покажет 4 варианта, можно любой)

**Вариант CNAME (для www):**
- Тип: `CNAME`
- Имя: `www`
- Значение: `cname.vercel-dns.com`

### 7.2. Настраиваем DNS у регистратора

Зайдите в личный кабинет вашего регистратора домена (Reg.ru, Timeweb, Beget и т.д.):

1. Найдите раздел **DNS-записи** или **Управление DNS**
2. Удалите все существующие A-записи и CNAME для `@` и `www` (если они есть)
3. Добавьте новые записи точно так, как показал Vercel
4. Сохраните

### 7.3. Ждём распространения (5–60 минут)

- Обычно домен начинает работать через 5–15 минут
- Максимум — до 1 часа (редко)
- Проверить статус можно на [https://dnschecker.org](https://dnschecker.org)

Когда домен подключится, Vercel автоматически выпустит бесплатный SSL-сертификат (HTTPS).

### 7.4. Перенаправляем www → без www (или наоборот)

В том же разделе Domains в Vercel вы можете настроить редиректы.

## Часть 8. Делаем письма красивыми (info@konsungrus.ru)

Сейчас письма приходят с адреса `onboarding@resend.dev` — это выглядит непрофессионально.

### 8.1. Подтверждаем домен в Resend

1. Зайдите в [https://resend.com/domains](https://resend.com/domains)
2. Нажмите **Add Domain**
3. Введите `konsungrus.ru`
4. Нажмите **Add**
5. Resend покажет **3 DNS-записи** (обычно 2 TXT + 1 CNAME)

### 8.2. Добавляем DNS-записи

Добавьте эти три записи в DNS вашего домена (у регистратора), точно так же, как вы делали для Vercel.

**Важно:** Это другие записи, не путайте с записями Vercel!

### 8.3. Ждём подтверждения (обычно 5–30 минут)

Когда статус домена в Resend станет **Verified** — можно продолжать.

### 8.4. Меняем отправителя в коде

1. Откройте файл:
   ```
   /Users/konsung/Desktop/Мои_проекты/konsung-rus/app/api/contact/route.ts
   ```

2. Найдите строку (примерно 80-я строка):

   ```ts
   from: 'Сайт Консунг Рус <onboarding@resend.dev>',
   ```

3. Замените на:

   ```ts
   from: 'Сайт Консунг Рус <info@konsungrus.ru>',
   ```

4. Сохраните файл.

### 8.5. Выкладываем изменение

```bash
cd "/Users/konsung/Desktop/Мои_проекты/konsung-rus"
git add .
git commit -m "Настроен красивый отправитель писем info@konsungrus.ru"
git push
```

Через 1–2 минуты Vercel автоматически пересоберёт сайт.

Проверьте форму ещё раз — теперь письма будут приходить с красивого адреса.

## Часть 9. Как обновлять сайт после запуска (рабочий процесс)

Это самая важная часть для вас в будущем.

### 9.1. Общий алгоритм (всегда один и тот же)

1. Вносите правки в файлы (см. `ИНСТРУКЦИЯ_ПО_РЕДАКТИРОВАНИЮ.md`)
2. Сохраняете файлы
3. Открываете терминал в папке проекта
4. Выполняете три команды:

   ```bash
   git add .
   git commit -m "Краткое описание, что поменяли"
   git push
   ```

5. Ждёте 60–120 секунд
6. Открываете сайт и проверяете

Готово. Никаких FTP, FileZilla, загрузки файлов вручную — всё автоматически.

### 9.2. Примеры хороших сообщений коммитов

- `Обновлены цены на тест-полоски`
- `Добавлен новый глюкометр Sejoy BG-750`
- `Исправлена опечатка в разделе О компании`
- `Добавлена новость про Всемирный день борьбы с диабетом`

### 9.3. Как посмотреть историю изменений

```bash
git log --oneline -10
```

### 9.4. Preview-деплои (для проверки перед продом)

Vercel автоматически создаёт тестовые версии при каждом Pull Request.  
Пока вы работаете один — можно просто пушить в main и проверять на проде (сайт меняется быстро).

## Часть 11. Troubleshooting — что делать, если что-то пошло не так

### Проблема: Сайт открывается, но показывает старую версию

**Решение:**
- Нажмите `Ctrl + Shift + R` (Windows/Linux) или `Cmd + Shift + R` (Mac) — жёсткая перезагрузка
- Или откройте в режиме инкогнито

### Проблема: Форма не отправляет письма

**Пошаговая диагностика:**

1. Откройте консоль браузера (`F12` → вкладка Console)
2. Заполните форму и отправьте
3. Посмотрите ошибки в консоли

**Частые причины и решения:**

- `RESEND_API_KEY is not set` → вы забыли добавить переменную в Vercel (Часть 5)
- Письмо ушло в Спам → добавьте `info@konsungrus.ru` в контакты
- Ошибка 500 → проверьте, что ключ правильный и не истёк

### Проблема: Фото или PDF не загружаются (404)

**Решение:**
- Убедитесь, что файлы лежат в `public/images/...` и `public/pdfs/...`
- После добавления новых файлов — обязательно сделайте `git add . && git commit && git push`

### Проблема: "This site can’t be reached" или белый экран

**Решение:**
1. Проверьте, что деплой прошёл успешно (зелёная галочка в Vercel)
2. Нажмите **Redeploy** вручную
3. Посмотрите логи деплоя (кликните на деплой → View Build Logs)

### Проблема: После изменения домена сайт не открывается

**Решение:**
- Подождите 30–60 минут
- Проверьте DNS на [dnschecker.org](https://dnschecker.org)
- Убедитесь, что вы удалили старые A-записи

### Проблема: Хотите откатить неудачное изменение

```bash
git log --oneline -5          # посмотрите историю
git revert <хэш_коммита>      # откатите один коммит
git push
```

Или попросите помощи — пришлите ссылку на проблемный деплой.

## Часть 12. Полезные команды терминала (шпаргалка)

| Что нужно сделать                    | Команда |

| Перейти в папку проекта              | `cd "/Users/konsung/Desktop/Мои_проекты/konsung-rus"` |
